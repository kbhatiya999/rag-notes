    6. Troubleshooting and Reference

    This page contains solutions to common problems, a summary of file locations, and a quick reference for common actions.

    Common Issues

    App Won't Start or Quits Immediately

    This usually means there's an error in the underlying script or configuration.

    1. Test the script directly: Open the Terminal app and run the script to see the actual error message: bash ~/LightRAG/lightrag-launcher.sh

    2. Read the error message. Common errors include:

      • command not found: lightrag-server: The installation failed or ~/.local/bin is not in your PATH.

      • Solution: Re-run the installation command: bash uv tool install "lightrag-hku[offline,observability]"

      • No such file or directory: '/Users/yourname/LightRAG/.env': The configuration file is missing or in the wrong place.

      • Solution: Make sure ~/LightRAG/.env exists.

      • Address already in use or Port already in use: Another application is using the port number (e.g., 8020) that LightRAG wants to use.

      • Solution: Open ~/LightRAG/.env and change the PORT setting to a different number, like 8030 or 9000. Save the file and restart the app.

    Can't See the .env File in Finder

    Files that start with a dot (.) are hidden by default on macOS.

    • Solution: In Finder, navigate to your ~/LightRAG/ folder and press ⌘ + Shift + . (Command + Shift + Period) to toggle the visibility of hidden files.

    Server Starts, but Browser Shows "Can't Connect"

    1. Check the Port: Look at the server app's window. The startup message says which port it's using (e.g., Uvicorn running on http://127.0.0.1:8020). Make sure you are using this exact address and port in your browser.
    2. Check for Errors: Look for any error messages in the server window that might have appeared after startup.

    File Location Reference

    Here's a summary of where all the important files for your server app are located:

    Item Location Description
    The Mac App /Applications/LightRAG Server.app The application you double-click to start.
    Settings File ~/LightRAG/.env All configuration (models, ports, API keys).
    Server Data ~/LightRAG/ The main folder for vector stores and other data.
    Documents to Index ~/LightRAG/inputs/ Place your PDF and text files here.
    Launcher Script ~/LightRAG/lightrag-launcher.sh The core script used by the Platypus app.

    Quick Action Reference

    Goal Action
    Start Server Double-click LightRAG Server.app in Applications.
    Stop Server Press ⌘ + Q in the app window.
    Access Web UI Open http://127.0.0.1:8020 in your browser.
    Change Settings Edit ~/LightRAG/.env and restart the app.
    Add Documents Drag files into the ~/LightRAG/inputs/ folder.
